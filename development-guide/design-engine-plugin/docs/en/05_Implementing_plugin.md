# 5. Implementing the Plugin

This chapter describes plugin specifications by properties and methods.

---

## 5.1. Properties

### 5.1.1. `design_engine`

```python
design_engine: str
```

The plugin name specified when calling the Layout Design REST API.

### 5.1.2. `request_params`

```python
request_params: list[int]
```

Indicates the [input data](a01_Args_to_request_design_Function.md) required by the underlying Design Engine for a layout design request.
Use Python [Enum](https://docs.python.org/ja/3/library/enum.html) to define a list of required items.
The definitions are in `RequestParams` under `layout-design-compose/layout-design/layout-design/src/common/request_params_enum.py`.
Input items not specified in this property are not passed to the plugin.

Import `RequestParams` as follows:

```python
from common.request_params_enum import RequestParams
```

---

## 5.2. Methods

### 5.2.1. `request_design`

```python
request_design(args: dict) -> str
```

Passes the provided input data to the underlying Design Engine and starts the design process.
Returns a design ID as a string. The design ID can be generated by the plugin or the Design Engine.

#### Arguments

- `args`  
  A dictionary containing input data for the Design Engine. See [Appendix 1. Arguments to request_design](a01_Args_to_request_design_Function.md).

#### Return Value

Returns the design ID (`designID`) as a string to identify the layout design request.

If an error occurs in the plugin/design engine, raise `PluginProcessException` or a custom exception that inherits from it. See [6. Exception Handling](06_Handling_Exceptions.md).

#### How to Test

Follow these steps to call the [2.1. REST APIs of the Layout Design](02_LayoutDesignFunctions.md#21-rest-apis-of-the-layout-design):

1. Prepare the [sample input data](../../samples/data/post_request_data.json) to be passed to the Layout Design Request.
2. Run the layout design request with the file from step 1 as the request body.  
  Example:
  ```shell
  curl -X POST -H "Content-Type: application/json" "localhost:8011/cdim/api/v1/layout-designs?designEngine=<design_engine>" -d @post_request_data.json
  ```
  - Replace `<design_engine>` with the plugin name from [5.1.1. design_engine](05_Implementing_plugin.md#511-design_engine).
3. Confirm the response includes a JSON design ID (`designID`).  
  Example:
  ```json
  {"designID": "ca71c4bb-7ba8-41ea-9707-35f46b675857"}
  ```

The `designID` returned by the REST API is a unique string generated by the plugin or the Design Engine.
Use this `designID` to call Get Layout Design Result.

### 5.2.3. `get_all_design`

```python
get_all_design(fields: list[str], limit: int, offset: int, sort_by: str, order_by: str, status: list[str]) -> dict
```

Fetches a list of layout design results from the underlying Design Engine according to the provided filters.
Returns the list of layout design results as a dictionary.

#### Arguments

- `fields`  
  Which parts of the result to include.  
  Allowed values: `design`, `conditions`, `procedures` (multiple allowed).
- `limit`  
  The number of results to return.  
  Default: 100
- `offset`  
  The starting index for retrieval.  
  Default: 0
- `sort_by`  
  Which field to sort by.  
  Allowed values: `startedAt`, `endedAt` (single value).  
  Default: startedAt
- `order_by`  
  Sort order.  
  Allowed values: `desc`, `asc` (single value).  
  Default: desc
- `status`  
  Filter by design status.  
  Allowed values: `IN_PROGRESS`, `COMPLETED`, `FAILED`, `CANCELING`, `CANCELED` (multiple allowed).

#### Return Value

Returns a dictionary containing the list that meets the criteria, the number of results returned, and the total count matching the criteria.

If an error occurs in the plugin/design engine, raise `PluginProcessException` or a custom exception that inherits from it. See [6. Exception Handling](06_Handling_Exceptions.md).

#### How to Test

Follow these steps to call the [2.1. REST APIs of the Layout Design](02_LayoutDesignFunctions.md#21-rest-apis-of-the-layout-design):

1. Run List Layout Design Results with your desired filters as query parameters.  
  Example:
  ```shell
  curl -X GET "localhost:8011/cdim/api/v1/layout-designs?designEngine=<design_engine>"
  ```
  - Replace `<design_engine>` with the plugin name from [5.1.1. design_engine](05_Implementing_plugin.md#511-design_engine).

2. Confirm the response includes a JSON list of design results.  
  Example:
  ```json
  {
    "count": 100,         # Number of results returned
    "totalCount": 200,    # Total count matching the criteria
    "designs": [          # List of layout design results
      {
        "status": "COMPLETED",
        "requestID": "17d9a80f-1a32-4f5a-9c50-b1c64c259f15",
        "startedAt": "2025-05-10T00:00:00Z",
        "endedAt": "2025-05-10T00:00:00Z",
        "design": {...},
        "conditions": {},
        "procedures": [...],
        "id": "ca71c4bb-7ba8-41ea-9707-35f46b675857"
      },
      ...
    ]
  }
  ```

Example with filters via query parameters:
```shell
curl -X GET "localhost:8011/cdim/api/v1/layout-designs?designEngine=<design_engine>&status=COMPLETED&limit=10&orderBy=asc"
```

The example above fetches the 10 oldest items with status COMPLETED.

### 5.2.4. `cancel_design`

```python
cancel_design(design_id: str) -> str
```

Sends a cancellation request for the layout design associated with the specified design ID to the underlying Design Engine.
Returns the resulting design status.

#### Arguments

- `design_id`  
  The design ID string returned by the layout design request.

#### Return Value

Returns the post-cancellation design status as a string.
The status transition and HTTP status codes returned by the Layout Design are summarized below.

| Current Status | Next Status | HTTP Status | Notes |
| --- | --- | --- | --- |
| IN_PROGRESS | CANCELING | 202 | Accepted |
| COMPLETED | Not allowed | 409 | Conflict |
| FAILED | Not allowed | 409 | Conflict |
| CANCELING | CANCELING | 202 | Accepted |
| CANCELED | CANCELED | 200 | OK |

If the current status is COMPLETED or FAILED, the design process has already finished (successfully or not), so the API returns 409 Conflict.

If an error occurs in the plugin/design engine, raise `PluginProcessException` or a custom exception that inherits from it. See [6. Exception Handling](06_Handling_Exceptions.md).

#### How to Test

Follow these steps to call the [2.1. REST APIs of the Layout Design](02_LayoutDesignFunctions.md#21-rest-apis-of-the-layout-design):

1. Run Cancel Layout Design with the target design ID in the path parameter.  
  Example:
  ```shell
  curl -X PUT "localhost:8011/cdim/api/v1/layout-designs/<designID>?action=cancel&designEngine=<design_engine>"
  ```
  - Replace `<design_engine>` with the plugin name from [5.1.1. design_engine](05_Implementing_plugin.md#511-design_engine).

2. Confirm the response includes the next design status.  
  Example:
  ```json
  {"status": "CANCELING"}
  ```

### 5.2.5. `delete_design`

```python
delete_design(design_id: str)
```

Requests deletion of the design result for the specified design ID from the underlying Design Engine.

#### Arguments

- `design_id`  
  The design ID string returned by the layout design request.

#### Return Value

None.

If an error occurs in the plugin/engine, raise `PluginProcessException` or a custom exception that inherits from it. See [6. Exception Handling](06_Handling_Exceptions.md).

#### How to Test

Follow these steps to call the [2.1. REST APIs of the Layout Design](02_LayoutDesignFunctions.md#21-rest-apis-of-the-layout-design):

1. Run Delete Layout Design Result with the target design ID in the path parameter.  
  Example:
  ```shell
  curl -X DELETE "localhost:8011/cdim/api/v1/layout-designs/<designID>?designEngine=<design_engine>"
  ```
  - Replace `<design_engine>` with the plugin name from [5.1.1. design_engine](05_Implementing_plugin.md#511-design_engine).

2. Confirm the API returns HTTP 204 NO CONTENT.

3. Call Get Layout Design Result again for the deleted design ID and confirm the API returns HTTP 404 NOT FOUND.
