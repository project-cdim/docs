# 7. ログ出力

構成案設計機能のログ出力に関する基本的な方針として、構成案設計と各プラグインで異なるロガーオブジェクトを使用して、異なるログファイルへログレコードを出力します。  
上記の方針によりプラグインのログ出力について、使用するロガーや出力するログレコードのフォーマットに制約事項はありません。  
以下にプラグインのログ出力に関する注意事項をまとめます。
- プラグインでPython標準の[ロガーオブジェクト](https://docs.python.org/3.14/library/logging.html#logger-objects)を使用する場合、ルートロガーは使用しないでください。
- プラグインのログファイルの出力先ディレクトリに制約はありませんが、構成案設計のログファイルの出力先と同じディレクトリ (デフォルト: `/var/log/cdim/`) を指定することを推奨します。

以下の表にPython標準のロガーオブジェクトに準拠したログレベルとそれぞれの方針をまとめます。

| レベル | 方針 |
| --- | --- |
| DEBUG | システムの動作状況に関する情報 |
| INFO | 処理の開始終了など、何らかの注目すべきものがあるとき |
| WARNING | エラーではないが正常でもない状態が発生した場合 |
| ERROR | DBの接続に失敗したなど、システムのある機能を実行できない場合 |
| CRITICAL | プログラムの続行が不可能な致命的な問題が発生した場合 |
